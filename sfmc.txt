    <h1>POC Plan: Unified Data & Visualization</h1>
    <p><strong>Project:</strong> Government Cloud Plus - Data Cloud Visualization<br>
    <strong>Focus:</strong> Simulating Marketing Cloud Growth & Google Analytics Integration</p>

    <div class="highlight">
        <strong>Objective:</strong> Demonstrate how Salesforce CRM data, Web Engagement (Mocked GA4), and Email Activity (Mocked Marketing Cloud) combine in Data Cloud to create a 360-degree view and power <strong>Standard Salesforce Dashboards</strong> without moving data.
    </div>

    <h2>I. Inventory & Capability Assessment</h2>

    <h2>II. The Mocking Strategy</h2>
    <p>Since live traffic sources are not yet connected, we will utilize the <strong>Data Cloud Ingestion API (or CSV Upload)</strong> to feed the system logs that mimic real-world scenarios.</p>

    <h3>A. Mocking Google Analytics (Web Traffic)</h3>
    <p>We will create a CSV file (e.g., <code>mock_web_traffic.csv</code>) representing user behavior.</p>
    <ul>
        <li><strong>Linkage:</strong> Using a <code>hashed_email</code> or <code>Client_ID</code> that matches a real Contact in CRM.</li>
        <li><strong>Data Points:</strong> <code>event_name</code> (page_view), <code>page_location</code> (/home, /newsletter), <code>timestamp</code>, <code>engagement_time</code>.</li>
    </ul>

    <h3>B. Mocking Marketing Cloud Next (Email Engagement)</h3>
    <p>We will simulate the native DMO writing process by uploading <code>mock_email_engagement.csv</code>.</p>
    <ul>
        <li><strong>Linkage:</strong> Using <code>subscriber_key</code> that matches a Salesforce Contact ID.</li>
        <li><strong>Data Points:</strong> <code>email_name</code>, <code>status</code> (Sent/Opened/Clicked), <code>event_date</code>.</li>
    </ul>

    <h2>III. Technical Execution Plan</h2>

    <h3>Step 1: Connect Real Data (CRM)</h3>
    <ul>
        <li>Connect Salesforce CRM to Data Cloud via the standard connector.</li>
        <li>Ingest <strong>Contact</strong> objects.</li>
        <li>Map to the <strong>Individual</strong> Data Model Object (DMO).</li>
    </ul>

    <h3>Step 2: Ingest Mock Data</h3>
    <ul>
        <li>Create Data Streams for Web and Email CSVs.</li>
        <li>Map Web CSV to <strong>Web Engagement DMO</strong>.</li>
        <li>Map Email CSV to <strong>Email Engagement DMO</strong>.</li>
    </ul>

    <h3>Step 3: Identity Resolution (The Core Value)</h3>
    <ul>
        <li>Configure <strong>Identity Resolution Rules</strong> to match CSV IDs to Salesforce Contact IDs.</li>
        <li><strong>Outcome:</strong> Data Cloud merges the records into a single "Unified Individual" containing CRM details + Web History + Email History.</li>
    </ul>

    <h3>Step 4: Insights & Segmentation</h3>
    <ul>
        <li><strong>Calculated Insight:</strong> Create metrics such as "High Intent Users" (e.g., Opened 3+ emails & visited pricing page).</li>
        <li><strong>Segmentation:</strong> Build a segment of users ready for outreach based on the mocked web traffic.</li>
    </ul>

    <h3>Step 5: Visualization (Reports & Dashboards)</h3>
    <p>Instead of copying data to CRM, we will use <strong>Data Cloud Reports</strong>.</p>
    <ul>
        <li><strong>Enable Feature:</strong> Go to Setup > Reports & Dashboards > Enable "Data Cloud Reporting".</li>
        <li><strong>Create Report Type:</strong> Create a new Report Type selecting "Unified Individual" joined with "Web Engagement" and "Email Engagement".</li>
        <li><strong>Build Dashboard:</strong> Create a Standard Salesforce Dashboard with the following components:
            <ul>
                <li><em>Chart 1:</em> "Top 5 Viewed Newsletter Articles" (Bar Chart).</li>
                <li><em>Chart 2:</em> "Email Open Rate vs. Web Visits" (Correlation Chart).</li>
                <li><em>Chart 3:</em> "High Intent Customers" (Table List).</li>
            </ul>
        </li>
    </ul>

    <h2>IV. Value Proposition for Stakeholders</h2>

    <ul>
        <li><strong>Native Visualization:</strong> We can view millions of engagement records directly in Salesforce Dashboards without buying Tableau or CRM Analytics.</li>
        <li><strong>Zero Copy Reporting:</strong> The data sits in Data Cloud but is viewable in Core. We do not bloat our Contact tables with millions of "Page View" records.</li>
        <li><strong>Unified Intelligence:</strong> Marketing data (Email) and Website data (GA4) are visualized side-by-side with CRM data for the first time.</li>
    </ul>
